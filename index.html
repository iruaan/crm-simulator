De onde vêm os dados?

Toda vez que um cliente faz uma compra, envia um e-mail ou preenche um formulário, essa informação é salva num banco de dados. Pense nisso como uma grande planilha onde cada linha é um cliente.

⚡
Detalhe técnico Os dados brutos ficam num banco PostgreSQL de produção (crm.customers). A Dito conecta via conector CDC ou full-refresh dependendo do volume e frequência configurados.

Como os dados são coletados?

A extração é como um caminhão que vai buscar os dados no armazém (banco de dados) e traz para cá. Antes de carregar, ele confere se tudo está no lugar certo — se as colunas têm os tipos certos, se nenhum campo obrigatório está faltando.

⚡
Detalhe técnico O extrator valida o schema recebido contra o schema esperado. Divergências de tipo (ex: STRING onde se esperava DATE) são logadas como warnings e tratadas na camada de transformação.

Schema é o "molde" dos dados: define quais campos existem e qual o tipo de cada um (texto, número, data…). Se o dado chega em formato diferente do esperado, pode quebrar análises — por isso checamos isso aqui.

Como os dados são limpos?

Imagine que você recebeu uma lista de contatos toda bagunçada — nomes em minúsculo, telefones sem formatação, campos vazios. A limpeza padroniza tudo isso para que as análises funcionem corretamente.

📄 O que é um arquivo .sqlx?
É um arquivo do Dataform que combina configuração e SQL. O bloco config{} define o tipo da tabela e metadados. O SELECT define as transformações. O Dataform compila isso em SQL puro para rodar no BigQuery.

Onde ficam os dados prontos?

Depois de limpos, os dados vão para o BigQuery — um banco de dados na nuvem do Google feito para consultas rápidas em grandes volumes. É de lá que relatórios, dashboards e análises de marketing são alimentados.

Filtrar por marca ?
🏷️ Por que filtrar por marca?
A Dito é multi-tenant: todas as marcas compartilham o mesmo dataset. Sem o filtro brand_id, sua query retorna dados de todas as marcas misturados — o que pode vazar informações entre clientes e tornar análises incorretas. Sempre filtre!
WHERE brand_id = '{{brand_id}}' AND _is_current = TRUE
